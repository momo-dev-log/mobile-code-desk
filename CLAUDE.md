# 運用ルール

## PRがマージ/クローズされた後の追加コミットについて

同じブランチに対するPRが一度マージ（またはクローズ）されると、GitHubは
そのPRを自動的に再オープンしない。そのため、マージ済みPRのheadブランチへ
さらにコミットを積んでも、それらは**どのPRにも紐づかない孤立コミット**に
なり、mainには取り込まれない。

- 修正コミットを既存PRへ追加したい場合は、まず対象PRが**まだopenである
  こと**を確認してから push する。
- 対象PRが既にmerged/closedの場合は、同じブランチからmainへの**新しい
  PR**を作成し直す。
- 「pushしたから反映されているはず」と判断する前に、対象PRの状態
  （open/closed/merged）と、mainの実ファイル内容を直接確認する。
